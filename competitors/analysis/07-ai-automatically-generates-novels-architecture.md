# AI-automatically-generates-novels 技术架构分析

## 一、项目概述

### 1.1 项目定位

AI 小说创作助手（AI-automatically-generates-novels）是一个面向 **网文工作室和个人作者** 的 AI 辅助写作生产力工具。其核心设计理念是"**提示词即架构**"——通过精心设计的三层提示词体系（大纲/章节/内容）和灵活的变量替换系统，将 AI 生成能力封装为可配置的创作流水线。

项目当前版本 v5.2，已有数百家工作室和个人作者使用，定位为**实用型生产力工具**而非研究型系统。

### 1.2 核心目标

- **低门槛接入**：支持任何主流大模型（ChatGPT、Claude、Gemini、DeepSeek、豆包、通义千问、文心一言、Ollama）
- **提示词驱动**：三层提示词体系 + 变量替换 + 右键菜单配置，用户可完全自定义创作流程
- **效率优先**：AI 批量迭代优化、拆书功能、快捷词条（Shift+L）、右键润色/扩写/去AI味
- **全功能集成**：思维导图、知识库、AI 自评分迭代、主题切换、导入导出

### 1.3 用户画像

| 用户类型 | 使用场景 |
|---------|---------|
| 网文工作室 | 批量生成小说、快速迭代优化 |
| 个人作者 | 辅助创作、灵感激发、文本润色 |
| 内容平台 | 拆书分析、风格学习 |

## 二、技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **后端** | Flask | 轻量 Python Web 框架 |
| **前端** | jQuery 3.6 + 原生 JS | 无前端框架，直接 DOM 操作 |
| **模板引擎** | Jinja2 | 服务端渲染 |
| **AI 接口** | OpenAI SDK（兼容协议） | 统一 API 调用 |
| **思维导图** | jsMind 0.4.6 | 可拖拽思维导图 |
| **存储** | localStorage | 纯浏览器本地存储 |
| **依赖** | 12 个 Python 包 | 极简依赖 |

### 2.1 双通道 AI 接口设计

```python
# /gen — 高质量模型接口（用于大纲/章节/正文生成）
API_ENDPOINT_1 = 'http://ssb.org.cn:10035/v1'  # Claude-3.5-Sonnet

# /gen2 — 低成本模型接口（用于AI迭代优化和拆书）
API_ENDPOINT_2 = 'http://ssb.org.cn:10099/v1'  # Qwen2.5-72B
```

双通道设计的核心思想：**生成用贵模型，迭代/拆书用便宜模型**，实现成本优化。

### 2.2 多模型适配器模式

项目通过 `app-xxx.py` 文件实现了多模型适配器：

| 文件 | 对接模型 |
|------|---------|
| `app-chatgpt.py` | OpenAI GPT 系列 |
| `app-claude.py` | Anthropic Claude |
| `app-gemini.py` | Google Gemini |
| `app-deepseek.py` | DeepSeek |
| `app-doubao.py` | 字节豆包 |
| `app-tongyiqianwen.py` | 阿里通义千问 |
| `app-wenxinyiyang.py` | 百度文心一言 |
| `app-ollama.py` | Ollama（本地部署） |

每个适配器文件只需替换为 `app.py` 即可切换模型，所有适配器保持统一的 OpenAI 兼容接口格式。

## 四、核心设计：三层提示词体系

### 4.1 大纲提示词（Outline Prompt）

```
输入变量：${background} ${characters} ${relationships} ${plot} ${style}

输出要求：
1. 故事梗概（500 字，突出爆点和冲突）
2. 核心主题（100 字）
3. 人物塑造重点（每个角色 200 字）
4. 主要情节脉络（转折点列表）
5. 爆点设计（3-5 个重大转折）
6. 高潮铺垫（层层递进策略）
7. 结局构思（200 字）
```

### 4.2 章节提示词（Chapter Prompt）

```
输入变量：${outline} ${background} ${characters} ${relationships} ${plot} ${style}

输出格式：###fenge 分隔的章节列表

每章包含：
- 核心看点 / 时间地点 / 出场人物
- 情节线索（引子→发展→转折→悬念）
- 感情线发展 / 人物刻画重点
- 场景氛围 / 伏笔设置
```

### 4.3 内容提示词（Content Prompt）

```
输入变量：${outline} ${chapter_outline} ${background} ${characters} ${relationships} ${plot} ${style}

创作要求：
- 多感官描写（视觉/听觉/触觉/心理）
- 对话要求（性格/潜台词/推进剧情）
- 场景描写（关键细节/氛围/节奏）
- 情感刻画（细腻/避免直白/细节烘托）
```

### 4.4 变量替换系统

| 变量 | 来源 | 用途 |
|------|------|------|
| `${background}` | 基础设置 | 背景设定注入 |
| `${characters}` | 基础设置 | 人物设定注入 |
| `${relationships}` | 基础设置 | 角色关系注入 |
| `${plot}` | 基础设置 | 核心剧情注入 |
| `${style}` | 基础设置 | 写作风格注入 |
| `${outline}` | 生成结果 | 大纲内容注入 |
| `${chapter_outline}` | 生成结果 | 章节细纲注入 |
| `${selected_text}` | 用户选区 | 右键菜单目标文本 |

变量替换在前端 JavaScript 中通过简单的 `String.replace()` 链式调用实现。

## 五、核心功能

### 5.1 创作流水线

```
Step 1: 基础设置（背景/人物/关系/知识库/风格）
  ↓
Step 2: AI 生成书名和简介（可选）
  ↓
Step 3: 生成大纲（/gen 接口，高质量模型）
  ↓
Step 4: 根据大纲生成章节细纲（###fenge 分隔）
  ↓
Step 5: 分割为独立章节
  ↓
Step 6: 逐章生成正文
  ↓
Step 7: 右键菜单优化（评分/润色/扩写/去AI味）
```

### 5.2 右键菜单系统

三套可配置的右键菜单，覆盖创作全流程：

**大纲右键菜单**（8 项）：
AI 评分 / 深化冲突 / 增加伏笔 / 完善人物动机 / 强化感情线 / 优化节奏 / 扩充细节 / 提升高潮

**章节右键菜单**（8 项）：
AI 评分 / 深化情节 / 强化冲突 / 优化结构 / 增加细节 / 完善对话 / 设置伏笔 / 强化感情

**正文右键菜单**（8 项）：
AI 评分 / 优化文笔 / 扩写对话 / 强化情感 / 添加细节 / 改写视角 / 去除说教 / 润色升华

每项菜单都包含完整的提示词模板，通过 `${selected_text}` 注入选中文本，通过其他变量注入上下文。

### 5.3 AI 自迭代优化

```
用户设定：目标分数 + 迭代次数 + 切割粒度（句子/段落/字符数）
  ↓
Step 1: 智能切割文本为可独立评分的片段
  ↓
Step 2: AI 对每个片段评分（/gen2 低成本接口）
  ↓
Step 3: 筛选低于目标分数的片段
  ↓
Step 4: AI 重写低分片段（/gen2 低成本接口）
  ↓
Step 5: 替换原文本 → 重新评分
  ↓
Step 6: 重复直到达到目标分数或迭代上限
```

### 5.4 拆书功能

将已有小说拆解为结构化数据：
- 提取世界观设定
- 提取角色关系
- 提取情节脉络
- 提取写作风格特征

拆解结果自动填入基础设置字段，可用于"学习"某本小说的风格后进行仿写。

### 5.5 思维导图

基于 jsMind 实现的可拖拽思维导图：
- 用于构建总纲和章节结构
- 支持提示词与思维导图结合
- 节点内容可直接用于生成

### 5.6 知识库系统

- 知识库数据存储于"基础配置 → 知识库"字段
- 不常用知识库时，该字段可直接填充相关剧情
- 知识库内容会被引用组装到提示词中
- JSON 格式存储，支持分类管理

## 七、亮点与创新

### 7.1 提示词即架构

本项目最大的创新在于将"系统架构"简化为"提示词配置"。用户不需要理解 Agent、图工作流、状态机等复杂概念，只需：
1. 填写 5 个基础字段
2. 编辑 3 套提示词模板
3. 配置 3 套右键菜单

就能构建一个完整的 AI 辅助写作系统。这种"**零代码可配置**"的设计极大降低了使用门槛。

### 7.2 双通道成本优化

/gen（高质量模型）用于核心创作，/gen2（低成本模型）用于 AI 迭代和拆书。这种"好钢用在刀刃上"的策略实现了质量与成本的平衡。

### 7.3 右键菜单驱动的交互设计

24 个预设的右键操作覆盖了创作的方方面面，用户选中任意文本即可进行针对性优化。这种"**选中文本 → 右键操作 → 预览确认 → 应用替换**"的交互流程非常直观。

### 7.4 多模型适配器模式

8 个 `app-xxx.py` 适配器文件实现了"**一个前端，任意后端模型**"的灵活架构。用户只需替换一个文件就能切换 AI 模型。

### 7.5 实用功能集成

- **快捷词条（Shift+L）**：解决大量重复性输入
- **AI 自评分迭代**：自动检测低质量片段并重写
- **拆书工具**：学习已有小说的风格和结构
- **思维导图**：可视化构建大纲
- **多标签页同步**：团队协作友好

## 八、局限性与改进空间

### 8.1 局限性

| 局限 | 说明 |
|------|------|
| **无 Agent 架构** | 所有生成任务都是单次 API 调用，无多 Agent 协作 |
| **无上下文管理** | 章节间无连贯性保证，无长篇记忆 |
| **纯前端存储** | localStorage 容量有限（~5-10MB），无数据库 |
| **无质量保障** | 缺少系统性的质量评估和反馈循环 |
| **代码质量** | 前端逻辑全部在 index.html 中（1089 行），无模块化 |
| **API 密钥硬编码** | app.py 中 API 密钥明文存储 |
| **无认证系统** | 任何人访问即可使用 |

### 8.2 对 Fiction Academy 的启示

1. **提示词工程价值**：精心设计的提示词可以大幅提升生成质量，三层提示词体系可直接借鉴
2. **右键交互模式**：选中文本→右键操作的交互方式用户体验好
3. **双通道成本优化**：好模型生成 + 便宜模型迭代的策略值得学习
4. **拆书功能**：分析已有作品的结构和风格是有价值的辅助功能
5. **需要增强**：Agent 协作、长篇记忆、质量保障、数据持久化、安全性
