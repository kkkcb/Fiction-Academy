# Goal 模式需求文档

## 一、核心概念

### 1.1 什么是 Goal 模式

用户给出一个**创作目标**（如"写完第三章"、"补全世界观"），系统自主完成以下流程：

```
用户目标
  ↓
拆解为子任务列表（可编辑）
  ↓
逐步执行，每一步：
  - 检查前置条件（已知信息够不够）
  - 发现缺失 → 反向提问用户确认
  - 执行子任务
  - 自检结果
  ↓
全部完成，汇总报告
```

与普通对话模式的区别：

| | 普通模式 | Goal 模式 |
|---|---|---|
| 交互方式 | 一问一答 | 用户给目标，AI 自主推进 |
| 任务拆解 | 无 | 自动拆解为步骤 |
| 缺失检测 | 靠用户自己发现 | AI 主动发现并提问 |
| 进度跟踪 | 无 | 实时显示进度条 |
| 结果检查 | 无 | 每步自检 + 最终汇总 |

### 1.2 "缺失"是什么

AI 要判断"缺失"，前提是它心里有一个**完整的模板**——一个小说项目在某个阶段"应该有哪些东西"。

#### 完整性检查框架

```
小说项目完整性清单
│
├── 世界观层
│   ├── 地理环境
│   │   ├── 大陆/世界的基本格局（有几块大陆、海洋分布）
│   │   ├── 故事主要发生地的详细描述
│   │   └── 特殊地理（秘境、禁地、遗迹等）
│   ├── 势力体系
│   │   ├── 主要势力/门派/国家列表
│   │   ├── 势力间关系（同盟/敌对/中立）
│   │   └── 势力等级/实力对比
│   ├── 力量体系
│   │   ├── 修炼/能力体系（境界划分）
│   │   ├── 战力天花板和地板
│   │   └── 特殊能力/禁忌之术
│   ├── 社会规则
│   │   ├── 货币/交易体系
│   │   ├── 等级/阶层制度
│   │   └── 禁忌/法律
│   └── 历史背景
│       ├── 重大历史事件
│       ├── 当前时代背景
│       └── 传说/预言
│
├── 角色层
│   ├── 主角
│   │   ├── 基本信息（姓名/性别/年龄/外貌）
│   │   ├── 性格特点（至少3个核心特质）
│   │   ├── 能力/实力定位
│   │   ├── 背景故事（出身/经历）
│   │   ├── 核心动机（想要什么/为什么）
│   │   └── 成长弧线（从X到Y的变化）
│   ├── 重要配角
│   │   ├── 与主角的关系
│   │   ├── 基本信息 + 性格
│   │   ├── 在故事中的功能（导师/对手/盟友/恋人）
│   │   └── 独立动机
│   ├── 反派
│   │   ├── 核心动机（不能只是"坏人"）
│   │   ├── 与主角的冲突根源
│   │   └── 实力/威胁等级
│   └── 角色关系网
│       ├── 主要角色间的关系图谱
│       └── 关系的动态变化
│
├── 剧情层
│   ├── 核心冲突
│   │   ├── 主线冲突是什么
│   │   ├── 冲突的起因
│   │   └── 预期的解决方式
│   ├── 故事结构
│   │   ├── 开端（引入/触发事件）
│   │   ├── 发展（升级/转折）
│   │   ├── 高潮
│   │   └── 结局
│   ├── 关键事件
│   │   ├── 必须发生的剧情节点
│   │   └── 事件间的因果关系
│   └── 支线
│       ├── 支线内容
│       └── 与主线的交汇点
│
└── 风格层
    ├── 叙事视角（第一人称/第三人称/全知）
    ├── 文风基调（轻松/沉重/热血/温馨）
    ├── 目标读者群
    └── 预期篇幅（短篇/中篇/长篇/网文连载）
```

#### "缺失"的判定逻辑

```
1. 用户说了 "张三是一个剑修"
   → 角色层·主角·能力：✅ 已有
   → 角色层·主角·基本信息：❌ 缺少年龄、外貌
   → 力量体系·境界划分：❌ 不知道剑修有几个境界
   → 角色层·主角·背景故事：❌ 不知道他怎么成为剑修的

2. 用户说了 "青云宗是天下第一宗门"
   → 势力体系·主要势力：✅ 青云宗
   → 势力体系·势力等级：✅ 天下第一
   → 势力体系·势力间关系：❌ 还有其他势力吗？
   → 势力体系·实力对比：❌ 第二是谁？差距多大？
   → 地理环境·发生地：❌ 青云宗在哪里？
```

## 二、功能设计

### 2.1 Goal 模式入口

在对话输入框上方增加 Goal 模式触发方式：

```
普通输入：直接输入消息（现有行为）
Goal 模式：输入 /goal + 目标描述

示例：
  /goal 写完第三章
  /goal 补全世界观
  /goal 设计反派角色
  /goal 梳理角色关系
  /goal 规划后续10章大纲
```

### 2.2 Goal 执行流程

```
用户输入 /goal 补全世界观
        ↓
┌─────────────────────────────────────────────┐
│  🎯 目标：补全世界观                           │
│                                               │
│  分析中... AI 正在检查当前项目状态              │
│  - 读取工作台已有条目                          │
│  - 读取对话历史中的设定信息                     │
│  - 对照完整性清单逐项检查                      │
└─────────────────────────────────────────────┘
        ↓
┌─────────────────────────────────────────────┐
│  🎯 目标：补全世界观                           │
│                                               │
│  📊 完整性分析：                               │
│  ✅ 地理环境 - 基础格局已设定                   │
│  ⚠️ 势力体系 - 只有青云宗，缺少其他势力         │
│  ❌ 力量体系 - 境界划分未定义                   │
│  ❌ 社会规则 - 完全空白                         │
│  ⚠️ 历史背景 - 提到过"上古大战"但无详情         │
│                                               │
│  📋 执行计划（共5步）：                         │
│  1. [待执行] 确认境界体系划分                   │
│  2. [待执行] 设计青云宗之外的势力               │
│  3. [待执行] 补全势力间关系                     │
│  4. [待执行] 设定社会规则（货币/等级/法律）      │
│  5. [待执行] 补充上古大战历史详情               │
│                                               │
│  [✏️ 编辑计划] [▶️ 开始执行]                   │
└─────────────────────────────────────────────┘
```

### 2.3 执行过程中的交互

每一步执行时，如果发现信息不足，进入**反向确认**环节：

```
▶ 执行步骤 1/5：确认境界体系划分
        ↓
┌─────────────────────────────────────────────┐
│  🎯 步骤 1/5：确认境界体系                     │
│                                               │
│  当前已知：主角是"剑修"，提到过"金丹期"         │
│                                               │
│  ❓ 需要确认：                                 │
│                                               │
│  境界体系的总体规模是？                         │
│  ○ 三境制（低/中/高）                           │
│  ○ 五境制（练气→筑基→金丹→元婴→化神）          │
│  ○ 九境制（完整的修仙体系）                     │
│  ○ 自定义：[________________]                  │
│                                               │
│  是否有超越常规境界的特殊存在？                  │
│  ○ 有（如仙人/神境）                           │
│  ○ 没有，最高就是常规境界                       │
│  ○ 不确定，后续再定                            │
│                                               │
│  [提交回答]                                    │
└─────────────────────────────────────────────┘
```

用户回答后，AI 继续执行：

```
▶ 执行步骤 1/5：确认境界体系划分
  ✅ 已确认：五境制 + 仙境
  📝 正在生成境界体系文档...
  ✅ 已保存到工作台：【世界观设定 / 境界体系】

▶ 执行步骤 2/5：设计其他势力
  📝 正在基于已有信息生成势力设定...
        ↓
┌─────────────────────────────────────────────┐
│  🎯 步骤 2/5：设计其他势力                     │
│                                               │
│  基于已有设定，建议以下势力：                    │
│                                               │
│  1. 丹鼎宗（炼丹为主，与青云宗同盟）            │
│     原因：修仙世界通常需要丹药支撑              │
│  2. 魔教（反派势力，与青云宗敌对）              │
│     原因：需要主要对手                          │
│  3. 天机阁（中立情报组织）                      │
│     原因：推动剧情的信息来源                    │
│                                               │
│  对每个势力的调整：                             │
│  [保留] [修改] [删除] [新增自定义]              │
│                                               │
│  [确认并继续]                                  │
└─────────────────────────────────────────────┘
```

## 三、预设 Goal 类型

### 3.1 内置目标模板

| Goal | 说明 | 检查维度 |
|------|------|---------|
| `/goal 补全世界观` | 检查世界观完整性，补缺 | 世界观层全部 |
| `/goal 完善主角` | 检查主角设定完整性 | 角色层·主角 |
| `/goal 设计反派` | 创建或完善反派 | 角色层·反派 |
| `/goal 梳理关系` | 梳理角色关系网 | 角色层·关系网 |
| `/goal 写完第N章` | 写完指定章节 | 剧情层 + 前文一致性 |
| `/goal 规划大纲` | 生成或优化故事大纲 | 剧情层全部 |
| `/goal 检查一致性` | 检查已有设定是否矛盾 | 全部 |
| `/goal 自定义:xxx` | 任意目标 | AI 自动判断 |

### 3.2 每种 Goal 的前置条件

```
/goal 写完第三章
  前置检查：
  ✅ 第一章和第二章是否已完成？
  ✅ 本章出场角色是否都有设定？
  ✅ 本章涉及地点是否已定义？
  ✅ 上一章结尾的悬念/状态是什么？
  ❌ 如果缺失 → 先补全，再写章节

/goal 补全世界观
  前置检查：
  ✅ 至少有一个角色（知道故事类型）
  ✅ 至少有一条对话历史（知道用户风格）
  ❌ 如果项目完全空白 → 先引导用户做基础设定
```

## 四、技术方案

### 4.1 后端：Goal 执行引擎

```
POST /api/projects/{id}/goal/start
Body: { "goal_type": "complete_world", "description": "补全世界观" }
Response: {
  "goal_id": "xxx",
  "analysis": { 完整性分析结果 },
  "steps": [ 子任务列表 ],
  "status": "planning"
}

POST /api/projects/{id}/goal/{goal_id}/confirm-plan
Body: { "steps": [ 用户编辑后的子任务列表 ] }
Response: { "status": "executing", "current_step": 1 }

GET /api/projects/{id}/goal/{goal_id}/status
Response: { 当前执行状态、进度、当前步骤的问题 }

POST /api/projects/{id}/goal/{goal_id}/answer
Body: { "step_index": 1, "answers": {...} }
Response: { 下一步或完成 }

POST /api/projects/{id}/goal/{goal_id}/execute-step
Body: { "step_index": 2 }
Response: { 执行结果、生成的内容、是否需要确认 }
```

### 4.2 后端：完整性检查引擎

核心是一个 prompt，输入当前项目的所有信息，输出完整性评分和缺失列表：

```
你是一位小说创作顾问。请检查以下小说项目的完整性。

【已有设定】
{工作台条目内容}
{对话历史中的关键信息}

请对照以下维度逐项检查，判断每项是"完整/部分/缺失"：

1. 地理环境（世界格局、主要地点、特殊地理）
2. 势力体系（主要势力、势力关系、等级对比）
3. 力量体系（境界/等级划分、战力范围）
4. 社会规则（货币、等级制度、法律禁忌）
5. 历史背景（重大事件、时代背景、传说）
6. 主角设定（基本信息、性格、能力、动机、成长弧线）
7. 配角设定（关系、功能、独立动机）
8. 反派设定（动机、冲突根源）
9. 角色关系网
10. 核心冲突
11. 故事结构
12. 关键事件链

返回 JSON：
{
  "dimensions": [
    {"name": "地理环境", "status": "complete|partial|missing", "detail": "已有世界格局但缺少具体地点描述"},
    ...
  ],
  "completion_rate": 0.35,
  "priority_gaps": ["力量体系完全空白", "反派未定义"]
}
```

### 4.3 前端：Goal 面板

在工作台区域新增 Goal 面板，与工作台/编年史并列：

```
[工作台] [🎯 Goal] [编年史]
```

Goal 面板展示：
- 当前目标的进度条
- 子任务列表（待执行/执行中/已完成）
- 当前步骤的交互区域（问题+选项）
- 完成汇总

### 4.4 Goal 执行策略

每一步的执行模式：

```
步骤类型 1：需要用户输入
  → 展示问题 + 选项/输入框
  → 等待用户回答
  → 继续下一步

步骤类型 2：AI 可以自主完成
  → 展示"正在执行..."
  → AI 生成内容
  → 展示结果供用户确认
  → 用户确认/修改后保存到工作台

步骤类型 3：需要多轮对话
  → 切换到对话模式
  → AI 在对话中引导用户
  → 对话结束后提取结果保存
```

## 五、实施阶段

### Phase 1：基础 Goal 框架
- `/goal` 命令解析和 UI 入口
- 完整性检查 prompt 和接口
- 计划展示（步骤列表 + 进度）
- 最简单的单步执行（展示结果 → 确认保存）

### Phase 2：交互式执行
- 步骤间的反向确认（复用现有待确认机制）
- 多步骤串联执行
- 中途暂停/继续/跳过
- 执行完成汇总报告

### Phase 3：智能规划
- 根据目标类型自动选择检查维度
- 前置条件检查和自动补全
- 步骤依赖关系（B 必须在 A 之后）
- 与知识图谱联动（图谱数据辅助完整性判断）

### Phase 4：高级 Goal
- `/goal 写完第N章`（多步骤长任务）
- `/goal 检查一致性`（跨条目矛盾检测）
- 自定义 Goal（任意描述，AI 自动规划）
- Goal 模板市场（预设常用目标）

## 六、与现有功能的融合

| 现有功能 | Goal 模式中的角色 |
|---------|------------------|
| **工作台条目** | Goal 的数据来源和存储目标（读取已有设定、写入新设定） |
| **待确认（❓）** | Goal 步骤中信息不足时，自动生成确认问题 |
| **📝 总结** | Goal 完成后，自动总结对话中的新信息 |
| **+ 添加** | Goal 生成的结果自动创建工作台条目 |
| **助理系统** | Goal 执行时自动选择合适的助理 |
| **知识图谱（规划中）** | 完整性检查的数据源，Goal 结果写入图谱 |
| **编年史** | `/goal 写完第N章` 完成后自动更新编年史 |
