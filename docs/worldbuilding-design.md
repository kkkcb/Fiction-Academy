# 世界观模板体系设计

## 一、目标

让用户有引导地构建完整世界观，使 AI 角色能基于世界观自然演绎，不需要频繁提问。

## 二、世界观要素清单

### 第一层：核心设定（必填）

| 分类 | 要素 | 引导问题 |
|------|------|----------|
| **世界基调** | 时代背景与风格 | 现实/奇幻/科幻/仙侠/架空历史？整体基调是黑暗、轻松、热血、悬疑？ |
| **地理环境** | 空间范围与地貌 | 故事发生在什么类型的世界？（大陆/星球/城市/异空间）有哪些主要区域？各区域特色？ |
| **核心规则** | 力量体系/科技树 | 这个世界的核心驱动力是什么？（魔法/科技/灵气/超能力/无特殊）规则边界和限制是什么？ |

### 第二层：社会结构（强烈建议）

| 分类 | 要素 | 引导问题 |
|------|------|----------|
| **政治体制** | 权力结构 | 谁在统治？帝制/共和/联邦/部落/企业？权力如何更替？ |
| **社会阶层** | 阶级与流动性 | 社会分层如何？底层能向上流动吗？什么决定了社会地位？ |
| **经济体系** | 资源与贸易 | 主要经济活动？核心资源是什么？货币体系？贫富差距？ |
| **法律禁忌** | 规则与禁忌 | 最严重的罪行是什么？有什么绝对不能做的事？法律对谁有效？ |

### 第三层：文化与信仰（建议）

| 分类 | 要素 | 引导问题 |
|------|------|----------|
| **宗教信仰** | 神明与信仰 | 有哪些宗教/信仰体系？神明是真实的吗？信仰如何影响日常？ |
| **文化习俗** | 风俗与禁忌 | 重要的节日/仪式？出生/成年/婚姻/死亡的习俗？有哪些禁忌话题？ |
| **语言文字** | 沟通方式 | 不同种族/地区有不同语言吗？文字系统？是否存在通用语？ |
| **教育知识** | 知识传承 | 普通人能受教育吗？知识是公开还是垄断？禁忌知识？ |

### 第四层：冲突与势力（建议）

| 分类 | 要素 | 引导问题 |
|------|------|----------|
| **主要势力** | 阵营与组织 | 有哪些重要组织/势力？它们的利益冲突是什么？ |
| **历史大事件** | 关键转折点 | 最近 100 年最重要的 3 件事？它们如何影响当下？ |
| **潜在矛盾** | 未解决的冲突 | 世界面临的最大威胁？哪些矛盾随时可能爆发？ |
| **军事力量** | 武装与战争 | 主要武装力量？战争形态？普通人如何受到影响？ |

### 第五层：日常生活（可选）

| 分类 | 要素 | 引导问题 |
|------|------|----------|
| **科技/魔法水平** | 日常应用 | 普通人能接触到的科技/魔法程度？生活便利程度？ |
| **交通通讯** | 出行与联系 | 人们如何出行？信息如何传递？距离感？ |
| **饮食居住** | 生活方式 | 普通人吃什么住什么？地区差异？ |
| **种族物种** | 智慧生命 | 除了人类还有哪些智慧种族？关系如何？ |

## 三、角色卡要素清单

| 要素 | 引导问题 |
|------|----------|
| **基本信息** | 姓名、年龄、性别、外貌特征 |
| **身份背景** | 出身哪里？职业/身份？在社会中的位置？ |
| **性格特征** | 核心性格（3个形容词）？说话风格？典型口头禅？ |
| **核心动机** | 最想要什么？最害怕什么？底线是什么？ |
| **能力特长** | 擅长什么？不擅长什么？特殊技能来源？ |
| **人际关系** | 最重要的3个人？与各方势力的关系？ |
| **内在矛盾** | 理想与现实的冲突？信念与欲望的矛盾？ |
| **与世界观的关联** | 如何被世界规则影响？在世界中的独特位置？ |

## 四、实现方案（待定）

### 数据结构
- 新增 `worldbuilding_templates` 表：存储模板分类和引导问题
- `workspace_items` 表中 `item_type` 新增子类型：`world_basic`（基础）、`world_society`（社会）、`world_culture`（文化）、`world_conflict`（冲突）、`world_daily`（日常）
- 角色卡结构化为 JSON 字段

### 前端 UI
- 引导式问卷：按层级逐步填写，每步有引导问题和示例
- 世界观总览：以卡片形式展示完整世界观
- 完成度指示：显示各层级的填写进度

### AI 辅助
- 基于已填写的信息，AI 可以建议补充哪些方面
- 自动检测世界观内部矛盾
- 生成世界观摘要供角色使用

## 五、后续：循环模式

当世界观足够完整后：
1. 用户创建角色（基于角色卡模板）
2. 设定任务目标（如"基于世界观和角色，生成第一章"）
3. 启动循环模式 → 角色基于自己的设定和世界观自然演绎
4. 世界观作为"宪法"，所有角色行为必须在其框架内
